# 弹窗主题（壁纸）方案（定稿）

## 1. 目标

将当前黑底提醒弹窗升级为可配置主题系统，支持：

- 背景（纯色 / 图片）
- 遮罩（颜色、透明度、渐变）
- 文字样式与排版
- 实时预览
- 预设复用与批量应用
- 会员能力预留（free / pro）

---

## 2. 信息架构（双层）

### 2.1 系统设置：主题工坊（重度编辑）

承载完整能力：

- 主题库
- 主题编辑器
- 批量应用

### 2.2 子项新建/编辑：轻量入口 + 可选详细编辑（非全屏）

默认仍轻量：

- 主弹窗主题选择
- 休息弹窗主题选择（`splitCount > 1` 时显示）
- 小预览（**未展开详细编辑时**：`ThemePreviewEditor`，支持双击改主文案、拖拽/旋转/缩放；变更即时 `updatePopupTheme`）
- 「主题工坊」跳转

**补充（已实现）**：主题下拉旁 **「编辑主题」** → 在**同一卡片下方内联展开** **`PopupThemeEditorPanel`**（`max-h` + 滚动），不再使用全屏 Portal；再点同一按钮为 **「收起详细设置」**（有未保存改动时确认）。保存策略见 `AGENTS.md` 4.11（独占则原地更新，有他项引用且内容变更则 fork 新主题并绑定 id）。

**分层排版**：`content/time/countdown` 可单独 **对齐 / 字间距 / 行高**（`PopupThemeEditorPanel`「当前选中层 · 排版」+ 主进程 `reminderWindow` 分层 CSS）。

---

## 3. 数据结构约定

- 主题为独立实体（建议 `popupThemes`）
- 子项仅绑定主题 id
  - `mainPopupThemeId`
  - `restPopupThemeId`

这样可避免样式字段在每条子项重复，便于预设复用和批量应用。

## 图层对象管理 V1（已定稿，待开发）

**目标**：在主题编辑中增加 **PS/Keynote 式对象列表**，选中对象后下方属性面板切换为该对象参数；遮罩、背景并入同一套图层模型，**首期不做**面板自由 Docking。

### 类型与数量（首期）

| 类型 | 数量上限 | 删除 |
|------|----------|------|
| **主提醒文案**（绑定层） | 1（永存） | 不可删 |
| **时间**（绑定层） | 1（永存） | 不可删 |
| **补充文本**（仅排版） | 5 | 可删 |
| **图片**（PNG 等） | 5 | 可删 |
| **遮罩** | 1 | 按产品约定（可与背景同属特殊槽位） |
| **背景** | 1 | 同上 |

「5 个文本」**不包含**主文案与时间，仅为补充装饰文案。

### 行为

- **显示/隐藏**：写入主题，**真实弹窗与预览一致**；隐藏 = 不绘制该层（非半透明镂空）。
- **兜底**：用户将所有层设为隐藏时，弹窗 **仅纯黑背景**（`#000`）。
- **排序**：图层列表 **上移/下移**（或拖拽排序），顺序 = **z 序**，`reminderWindow` 输出顺序须与预览一致。
- **UI**：图层栏在文字/参数区 **上方**；支持 **显示/隐藏图层栏**、**折叠**列表区域。

### 技术落点（实现时自检）

- `PopupTheme` 从扁平字段演进为 **`layers: PopupLayer[]`（或并存 + normalize 迁移）**；`formatVersion` 递增。
- `main/settings.ts` **`normalizePopupTheme`**（或等价）必须可逆迁移旧 JSON。
- **`reminderWindow.ts`**：按序生成 DOM；图片仍避免 `data:` 超长 URL，遵守临时文件 `loadFile`。
- **预览**：`ThemePreviewEditor` Moveable 目标改为 **按 layerId**；绑定层禁止删除，可选 **锁定** 误操作。
- **休息结束倒计时**：仍为 **独立 HTML**，不走主题图层（与现网一致）。

### 首期明确不做

- 编辑区各面板 **自由拖拽布局**（Docking）。
- 动态壁纸等扩展类型（模型预留 `type` 即可）。

## 5. 批量应用规则

点击“应用所有”后，弹确认框：

- 作用范围（默认全选）
  - 所有闹钟
  - 所有倒计时
- 应用目标
  - 主弹窗主题
  - 休息弹窗主题

约束：

- 作用范围至少选择一个
- 应用目标至少选择一个
- 显示预计影响条目数量

## 6. 会员预留

建议字段：

- `entitlements.popupThemeLevel = 'free' | 'pro'`

建议门控：

- free：纯色/单图/基础文字
- pro：文件夹壁纸、渐变高级参数、排版高级能力

UI 可先显示锁定态提示“会员功能”，后续再接实际鉴权/支付。
